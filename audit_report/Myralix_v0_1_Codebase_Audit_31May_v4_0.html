§3.7 · The percentages, in detail

What the percentages mean — closing each gap to 100%

Fair challenge raised at the meeting: "if Sourcing is 85%, what is the missing 15%?" The percentages in §3.6 were directional scope-judgement estimates, not item-by-item counts. This section is the rigorous breakdown — for every area, here's what's done, what's missing, and how much effort closes the gap. "100% for v0.1" means the Module Guide v0.1 scope, not every conceivable feature; items the Module Guide explicitly defers to v0.2 do not count against 100%.

3.7.1 The framing — three categories of "missing"

For each remaining gap, items fall into one of three categories:

  1. To 100% within v0.1 scope — Module Guide explicitly lists this as in-scope for v0.1. These count against the percentage. Effort estimated in hours/dev-days.
  2. Decision-dependent — Module Guide lists it for v0.1 but the team is considering deferral (e.g., SC-03 Q&A, SO-02 Outreach). Decisions D-1/D-2 in §8 resolve these.
  3. Out of v0.1 scope (deferred to v0.2) — Module Guide explicitly defers these. They do NOT count against the percentage. Listed for completeness.

9.1 Glossary for non-technical readers

ATSApplicant Tracking System — the software a recruiter uses to manage hiring end-to-end.
Multi-tenancySupporting multiple separate companies on one install without them seeing each other's data.
JWTA small encrypted token the browser sends with every request to prove identity. Now includes company_id.
RBACRole-Based Access Control. Admins do more than Staff.
MigrationA SQL file that adds new tables or columns. Run once when DB is set up.
Module / Service / ControllerLayers of backend code: route handles URL, controller decides action, service does business logic, model talks to DB. Standard pattern.
SSEServer-Sent Events — AI streams its response word-by-word instead of waiting for full answer.
RPARobotic Process Automation — headless browser that logs into Seek/LinkedIn as the recruiter, clicks like a human.
RubricThe set of weighted criteria AI uses to score CVs against a job. Configurable per job.
4-Level WorkspaceThe architectural pattern every workflow module follows: L1 Workboard · L2 Position · L3 Candidate · L4 Calibration.
StepperThe UI pattern where a workspace has 3-5 horizontal stages you click through. Used by every L3 Candidate view.
BullMQA Redis-backed job queue for Node.js. Used for AI calls, RPA, vendor adapters — anything that shouldn't block an HTTP request.
compensation_engineBackend service that computes Indonesian salary statutory rollup: PPh21 + PTKP + BPJS Kesehatan + BPJS Ketenagakerjaan + Gross/Nett toggle. Single source of truth, never computed in frontend.
PKWTT / PKWTIndonesian labour contract types. Permanent (PKWTT) vs Fixed-term (PKWT). Day-90 onboarding decision chooses between them.
UU PDPIndonesia's data privacy law (UU 27/2022). Background checks must collect explicit candidate consent.
UU 13/2003 §60Indonesian labour law clause governing probation period (max 3 months) and the Day-90 decision (Konversi PKWTT / Perpanjangan / Pemutusan).
SKCKSurat Keterangan Catatan Kepolisian — Indonesian police record clearance. One of the BG Check lanes.
SIPPSurat Izin Praktik Psikologi — psychologist practice license. Stored on psikolog profile per Module Guide §9.
